Core Capabilities

OrcasAgent™ coordinates the work.

OrcasAgent™ is not a single-purpose assistant. It is the orchestration agent responsible for coordinating enterprise agent work across goals, knowledge, tools, model choices, Dockets, evaluations, evidence, and operating boundaries.

  • Interprets enterprise intent: Converts high-level business goals into sequential code execution strategies.
  • Coordinates specialist agents: Spawns micro-agents for narrow operational targets to minimize cognitive error rates.
  • Connects work to Dockets: Enforces block dockets on all cloud transformations.
  • Supports model routing decisions: Evaluates provider availability, compliance, and latency metrics before triggering LLM calls.
  • Tracks evidence and readiness: Ensures every step has documented sandbox validations before deployment.
  • Escalates blockages: Safely yields and notifies SRE teams when risk tolerances or approvals require manual intervention.

âš™ Chamber 2: Operating Sequence & Flows
Controlled Execution Sequence

How governed agent work moves through NSAM.

OrcasAgent™ follows a controlled operating sequence. Work starts with intent, gains context from knowledge, is coordinated through agents and models, is governed through Dockets, is validated through proof, and is measured through evidence and outcomes.

01. Capturing and Scoping Intent

Every operational sequence begins with clean business or technical goals expressed in plain natural language. OrcasAgent™ parses the input text, establishes constraints, and identifies target success metrics.

  • What Happens: Core objectives and boundaries are formalized.
  • OrcasAgent™ Coordinates: Scope checks, user verification, and initial risk rating.
  • Evidence Created: Immutable Intent Payload, tagged with FDE signature.
// EXPECTED PAYLOAD // 
{
  "intent": "Optimize DB storage",
  "restrictions": [
    "No schema modifications",
    "Limit to zone us-central"
  ],
  "triggered_by": "FDE-USER-102"
}

02. Gathering Semantic Knowledge

To ground actions accurately, OrcasAgent™ retrieves contextual information from internal repositories, design standards, and system topologies. It loads corporate guidelines and SRE runbooks into the session context.

  • What Happens: Semantic grounding maps current policies and operational dependencies.
  • OrcasAgent™ Coordinates: Vector database searching, graph node matching, and validation check.
  • Evidence Created: Context Node Reference Logs in Second Brain.
// INTEGRATED KNOWLEDGE //
MySQL_Replication_Lag_Manual_v2 Lineage: gs://company-runbooks/sre/mysql-replica.pdf

03. Multi-Agent Orchestration

Rather than delegating all commands to a single model, OrcasAgent™ acts as the orchestra conductor, spawning specialized sub-agents (e.g. CLI runner, code analyzer, budget checker) and assigning narrow roles.

  • What Happens: Sub-agents are loaded and assigned discrete execution scopes.
  • OrcasAgent™ Coordinates: Handshakes, prompt constraints, and message buses between sub-agents.
  • Evidence Created: Agent Execution Manifest within the session state.
[Orchestrator] Active
↳ Spawning Specialist Agent: SQL-CLI-Runner
↳ Spawning Specialist Agent: Cost-Validator

04. Block Docket Governance

Before any action is taken on any cloud target, a formal Docket is compiled. This document contains the step-by-step action plan, safety constraints, risk labels, and required manual approval signatures.

  • What Happens: Execution plan is frozen and reviewed for security gates.
  • OrcasAgent™ Coordinates: Pre-approval checks, Slack gate escalation, and signature validation.
  • Evidence Created: Sealed Block Docket, tagged as PENDING_APPROVAL.
● DOCKET_SEALED_GATE_PENDING
Docket ID: DK-SRE-8913 Manual Signature Required

05. Sandboxed Dry-Run Proof

To verify the generated command string matches SRE standards, OrcasAgent™ executes a simulated run inside an isolated sandbox environment. The runtime captures output logs and screens to prove safety bounds.

  • What Happens: Simulated code execution proves validation parameters.
  • OrcasAgent™ Coordinates: Sandbox container allocation, log monitoring, and validation checks.
  • Evidence Created: Execution Sandbox Verification Log.
// SANDBOX EMULATOR // 
[INFO] Initializing sandboxed MySQL...
[INFO] Executing: 'RESET SLAVE;'
[OK] Exit code: 0
[ASSERT] No database mutations on master - PASSED

06. Controlled Execution & Audit

Once all validations and approvals pass, the Docket is promoted to the production lane. The runtime records the operational results and traces every action's lineage within the Second Brain graph.

  • What Happens: Scoped actions are executed on target infrastructure.
  • OrcasAgent™ Coordinates: API connection, transaction commit, and final status reporting.
  • Evidence Created: Production Outcome Node & Cryptographic Docket Signature.
